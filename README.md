## 👁️ Overview

**Debunk & Bet** is a comprehensive web platform and browser extension designed to strip emotion and noise from prediction markets. By leveraging real-time context gathering and verifiable edge-computing LLMs, we provide users with mathematically sound probabilities and optimized bet sizing specifically tailored for binary markets. 

Whether you interact through our dedicated web application or our seamless browser extension, your goal is the same: to find the "edge" against the crowd using cryptographic, AI-driven evidence.

## 🏗️ Architecture: Public vs. Private

To ensure transparency while protecting our intellectual property, Debunk & Bet employs a bifurcated repository architecture:

- **The Public Repository (This Repo):** Contains the user interface (web app, extension components) and the public API gateways. This allows developers to audit the client-side execution, verify UI integrity, and seamlessly connect to our services.
- **The Edge Protocol (Private Repository):** The proprietary backend engine driving the core intelligence of Debunk & Bet. It handles the heavy computational lifting, mathematical transformations, and secure AI agent orchestration.

## 🧠 How "The Edge Protocol" Works (The Brains)

The magic behind Debunk & Bet lives inside **The Edge Protocol**. To build trust with our community of quants and developers, here is an inside look into our proprietary inference logic.

### 1. Dual Agent Research
Before probability can be established, evidence must be weighed fairly. We utilize a **Dual Agent architecture** to synthetically recreate adversarial debate:
- **Agent YES**: Scours real-time news sources and indices looking exclusively for arguments, data, and context supporting the event's occurrence.
- **Agent NO**: Operates entirely independently to fetch counter-evidence, establishing a competitive base rate for why the event will *not* occur.

### 2. Verifiable Inference (OpenGradient TEE)
Once the evidence is collected, it is submitted to our "Judge" LLM. To guarantee the Judge operates without interference, all inference runs inside a **Trusted Execution Environment (TEE)** on the Sepolia network via **OpenGradient**. 

Every analysis generates a **cryptographic proof** and an on-chain transaction hash. This ensures that the Likelihood Ratios generated by the Judge were derived from the exact prompt and exact model stated, making our AI fully verifiable.

## 📐 The Mathematics (The Quant Section)

Our engine is built on robust Bayesian statistics modified for real-world epistemological uncertainty (v5.2 Binary Markets math). 

### Asymmetric Likelihood Ratios
Instead of generating a raw probability, our Judge evaluates the evidence from both agents to produce two distinct Likelihood Ratios ($\text{LR}_{yes}$ and $\text{LR}_{no}$). The final composite Likelihood Ratio is asymmetric:

$$
\text{Final\_LR} = \frac{LR_{yes}}{LR_{no}}
$$

### 1D Bayesian Update in Odds Space
We map the crowd's current probability to prior odds ($O_{prior}$), and update our belief using the calculated $\text{Final\_LR}$ to find the posterior odds ($O_{post}$):

$$
O_{post} = O_{prior} \times \text{Final\_LR}
$$

*From $O_{post}$, we derive our internal true probability.*

### Risk Management & The Epistemic Penalty
Finding an edge is only half the battle; managing risk is the other. We employ a **modified Kelly Criterion** for bet sizing. 

Crucially, when Agent YES and Agent NO both return extremely strong, yet sharply conflicting evidence, the system detects an **"Informational War."** In these scenarios of high epistemological uncertainty, we apply an **Epistemic Penalty**—mathematically shrinking the recommended bet size (confidence penalty) to protect capital against unknown unknowns.

## ⚙️ Local Setup & Usage

To run the public API gateway and UI locally:

1. **Clone the repository:**
   ```bash
   git clone https://github.com/your-org/debunk-and-bet.git
   cd debunk-and-bet
   ```

2. **Set up the virtual environment & install dependencies:**
   ```bash
   python -m venv venv
   source venv/bin/activate
   pip install -r requirements.txt
   ```

3. **Configure Environment Variables:**
   Rename `.env.example` to `.env` and fill in your OpenGradient API keys.
   ```bash
   cp .env.example .env
   ```

4. **Launch the FastAPI Server:**
   ```bash
   uvicorn app.main:app --reload
   ```
   The application will be available at `http://localhost:8000`.
